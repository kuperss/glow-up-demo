## 上傳到 NotebookLM 的步驟

### 1. 準備
- 開啟 [https://notebooklm.google.com](https://notebooklm.google.com)
- 用「擁有舞光戰將 KB notebook 那本筆記本」的 Google 帳號登入
- 進入 Notebook ID `1af7e026-a5e0-443e-81e7-87c09ba07a6d` 那本

### 2. 上傳每份檔案
點 **Add source** → 選 **Upload Source** → 拖入 5 份 `.md` 檔案

### 3. 命名
NotebookLM 會用檔案名作為 source 名稱。已用 `01_` `02_` 開頭排序，會按章節順序顯示。

### 4. 等候索引
NotebookLM 會自動切片、嵌入、索引，每份約 30 秒。完成後即可問答。

### 5. 測試 RAG 效果
回到舞光戰將訓練網站，到 admin → AI 設定 → 測試呼叫，問：
- 「索爾崁燈跟奧丁崁燈差在哪？」
- 「客戶嫌貴怎麼處理？」
- 「車禍了第一步要做什麼？」
- 「久任 10 年可以拿多少獎金？」

NotebookLM 應該會基於上傳的 5 份 .md 回答，並在答案後附引用標號 [1] [2] ...

## 維護與更新

### 平常更新內容（最常見）
1. **直接編輯 .md 檔案**（用任何文字編輯器）
2. 到 NotebookLM 的同一本 notebook
3. 點該份舊 source → **Delete**（或 **Re-upload**）
4. 重新上傳新版 .md
5. 等 30 秒索引完，前端 AI 立即用新版

> 不需要重新部署 fly.io 後端 — NotebookLM 是即時生效的。

## 常見問題

### Q: 為什麼要拆 5 份而不是 1 份大檔？
A: NotebookLM 對「來源」的處理會分別檢索。拆開後：
- AI 答覆能精準引用「來自第 X 份的 Y 段」
- 維護時只需要動到對應檔，不影響其他內容
- 上傳速度更快（每份 < 30 秒索引）

### Q: 可以改成 JSON 嗎？
A: 可以，但 Markdown 對 NotebookLM RAG 更友善：
- Markdown 有自然的標題層級（`#`/`##`/`###`），NotebookLM 自動建索引
- JSON 需要 NotebookLM 自己 parse 結構，效果較差
- 如果未來要從程式批次更新（例如從 ERP 導出），再考慮 JSON

### Q: 內容跟訓練網站 HTML 一致嗎？
A: 是同源內容，只是格式不同。HTML 教材給人看（含動畫、互動），這份 .md 給 AI 看（純文字、結構化）。
未來改一邊另一邊也要同步，不然會有不一致風險。

### Q: 上傳後 AI 第一次回答還是慢，正常嗎？
A: 正常。NotebookLM RAG 的單次查詢約 10-25 秒（含檢索 + 生成）。比直接 LLM API 慢但答案有來源引用 + 不會編造。

### Q: AI 答出範圍外的問題怎麼辦？
A: 到 admin → AI 設定 → 系統提示詞，加強範圍限制。建議加：
> 你只能回答展晟舞光相關問題。範圍外的禮貌拒答：「這個問題不在我的訓練範圍內」
