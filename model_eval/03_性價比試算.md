# 性價比試算

> 把 [02_評分表.md](02_評分表.md) 的品質分數 + token 用量,換成實際成本,選出 CP 值最高的模型。

---

## 核心公式

```
每千次對話成本(USD)
  = (平均 input tokens × input 單價 + 平均 output tokens × output 單價) × 1000
  ÷ 1,000,000

性價比 = 品質總分 / 每千次對話成本
```

**性價比數字越大越好**(品質高 + 成本低)。

## 步驟 3 · 算每千次對話成本

每千次對話成本 = (input × input 單價 + output × output 單價) ÷ 1000

> 提醒:單價是「每 1M tokens 多少 USD」,所以實際公式是
> `(input_tokens × input_price + output_tokens × output_price) × 1000 / 1_000_000`
> = `(input_tokens × input_price + output_tokens × output_price) / 1000`

## 步驟 5 · 決策(品質下限 + 成本上限)

純看「性價比」會被便宜 + 還能看的模型騙到。建議再套兩個篩選條件:

### 品質下限(不能低於這個分,不然體驗會差)
- 建議:**品質總分 ≥ 120 / 150**(80%)
- 或加權分 ≥ 144 / 180(80%)
- 任何一題拿 1 分(災難級)直接淘汰,不管總分多高

### 成本上限(預算不能爆)
- 預估你每天會跑幾次對話(例:50 個業務新人 × 平均 10 題 = 500 次/天)
- 一個月 ≈ 15,000 次 = **15** 個千次
- 算月成本上限,看哪些模型在預算內

## 範例(以 2026 中現行 5.4 / 5.5 系列真實價格)

> 單價以 https://developers.openai.com/api/docs/pricing 為準,可能調整。
> token 量假設:**input 22,000**(舞妞 + 5 KB)+ **output 300**(200 字內回應)。
> 品質分數為示意,**請以你實際打的分數為準**。

### A. 冷啟動千次對話成本(每次都重算 input,沒走 cache)

| 模型 | 品質(假設 /150) | input USD/1M | output USD/1M | 千次成本 USD | 性價比 |
|---|---|---|---|---|---|
| gpt-5.5 | 145 | 5.00 | 30.00 | 119.00 | 1.22 |
| gpt-5.4 | 138 | 2.50 | 15.00 | 59.50 | 2.32 |
| gpt-5.4-mini | 124 | 0.75 | 4.50 | 17.85 | 6.95 |
| **gpt-5.4-nano** | 116 | 0.20 | 1.25 | **4.78** | **24.27** |

### B. Cache hit 千次對話成本(實際情境 — input 打 1 折)

你的 system prompt(舞妞 + 5 KB)固定不變,連續呼叫走 prompt cache,**input 單價約 0.1 倍**:

| 模型 | input cached USD/1M | 千次成本 USD | 換算 NTD(× 32) |
|---|---|---|---|
| gpt-5.5 | 0.50 | 11.30 | NT$362 |
| gpt-5.4 | 0.25 | 5.90 | NT$189 |
| gpt-5.4-mini | 0.075 | 3.00 | NT$96 |
| **gpt-5.4-nano** | 0.02 | **0.82** | **NT$26** |

**怎麼讀:**
- 一旦走 cache,nano 一千次對話只要 NT$26 — 比 mini 省 3.6 倍、比旗艦 5.5 省 14 倍
- **但這建立在「nano 品質夠用」上** — 反幻覺、結構化查找這幾個維度若翻車,省的錢遠補不回錯誤回答的代價
- 業務新人沒能力判斷答案對錯,**正確性 > 便宜**

→ **決策原則**:
1. nano 6 題評分都過關(任一題不能拿 1 分)→ 採用 nano
2. 任一題翻車 → 升 mini(或補強 KB 後再測一次)
3. 多題翻車 → 直接升 mini,不要硬撐
